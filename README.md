# SPEngine
Типа движок на ps1 с использованием PSn00bSDK. 

# 🛠 КАК ВЫГЛЯДИТ РАБОТА
Для работы с движком придётся переписывать сам движок. Это вам не Юнити: открыли папку и скомпилировали. Здесь код — это ваш редактор.
Но тут достаточно понятный API (наверное, оценивал лично Я) и защита от дурака... полностью отсутствует.

Весь движок устроен по принципу «ну, столько-то хватит». При оптимизации размеры массивов уменьшаются вручную в коде, а при нехватке — соответственно повышаются. Сделано это ради ультимативной цели: урезать использование динамической памяти к минимуму и не ловить фрагментацию RAM на консоли.

**Так как работать:** 
Создал файлы сцены ➡️ заинклудил в `list.hpp` ➡️ указал `.cpp` файл в `CMakeLists.txt` ➡️ написал `cmake --build build` ➡️ профит!
*А, и не забудьте перед работой ввести `cmake --build build`.*

# 🚀 ЧТО ТУТ ИНТЕРЕСНОГО ПОД КАПОТОМ
Если вам лень ковырять код, вот ключевые технические фичи:
* **ECS на X-Macros (Registry):** Компоненты хранятся в плоских массивах (Structure of Arrays). Никаких тяжелых ООП-структур и виртуальных таблиц.
* **Storage с дедупликацией:** Хранилище для хранения данных между сценами. Проверяет файлы через `memcmp` перед загрузкой. Если ресурс уже в памяти — новый кусок RAM не отжирается.
* **Прямой Scratchpad:** Быстрая память процессора привязана через указатели прямо к физическому аппаратному адресу `0x1F800000`. Ноль оверхеда в рантайме.
* **Fx Оверлорд:** Эффекты переходов нагло перехватывают цикл `for(;;)` и генерируют геометрию эффекта силами самого ECS-реестра сцены, бесшовно подменяя старую сцену на новую прямо на пике анимации.
